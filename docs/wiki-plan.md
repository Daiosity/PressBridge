# Wiki Plan

This is a practical GitHub wiki structure for PressBridge.

It is meant to help developers evaluate and adopt the product without widening the promise beyond the current beta scope.

## 1. What Is PressBridge

- short product definition
- WordPress -> PressBridge -> React model
- who it is for
- who it is not for

## 2. Quick Start

- install plugin
- configure frontend URL
- run starter frontend
- verify route resolution
- test preview locally

## 3. How PressBridge Works

- route resolution model
- preview token model
- starter frontend roles
- safe public handoff model

## 4. Supported Beta Scope

- what core currently handles reliably
- what is beta-safe with caveats

## 5. Limitations And Non-Goals

- no theme parity promise
- no WooCommerce core guarantee
- no ACF or Elementor core guarantee
- no third-party/custom block compatibility promise
- no interactive WordPress-frontend-JS block guarantee

## 6. Validation And Scenario Guardrails

- route scenario matrix
- Gutenberg scenario matrix
- preview scenario matrix
- package/install validation
- how to run the validators locally

## 7. Extending Or Customizing Later

- how the starter can be customized
- how advanced compatibility should be framed separately from core
- where future docs for WooCommerce, ACF, or foundations would live
